请问变量a会被GC回收么，为什么呢?

不会
由于eval会产生欺骗词法作用域，因为eval可以在当前作用域声明变量，是动态解析的，浏览器并不知道 里面有没有声明变量 所以不会回收a

这样就会把a回收，因为eval是相对于全局的，已经和当前作用域没关系了

log出输什么

代码会报错，因为第三个log会报语法错误，
如果把第三个log注释，
第一个log输出 p.a: a, 先进行自身查找 obj.__proto__ -> Person.prototype， 如果没有再进行上一级查找 Person.prototype.__proto__ -> Object.prototype
第二个log输出 a，因为1.会把数字转成浮点型，此时是一个数字对象，然后用.操作符获取对象的属性，这是在原型链上查找, 最终查到到 Object.prototype，发现a属性就返回
